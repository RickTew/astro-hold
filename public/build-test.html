DISCARDED SUGGESTIONS (don't apply to a DOM HUD)

  • TextGeometry / FontLoader — 3D text only. Our title is HTML text; CSS text-shadow + animated gradient gives the same visual without the rebuild.
  • BoxGeometry / PlaneGeometry buttons — would require rebuilding the HUD inside the WebGL scene. Our tiles already render as DOM elements with hover states.
  • SpotLight / RectAreaLight banner sweep — Three.js lights don't illuminate DOM. The "light sweep" effect is the CSS-equivalent above.
  • UnrealBloomPass — postprocessing only affects the WebGL canvas, not DOM elements floating over it. CSS filter: drop-shadow is the practical equivalent.
  • material.wireframe = true — only valid on 3D mesh materials. "Wireframe" toggle above uses dashed CSS borders + ghosted icons as a holographic stand-in.
  • Group.position.lerp() for menu slide-in — CSS keyframes handle DOM slide-in natively, no need for a Three.js animation system.
  • Procedural GLSL noise shader — replaced with SVG feTurbulence, which is the W3C-standard procedural noise filter for DOM content. Same visual outcome, no shader pipeline.
